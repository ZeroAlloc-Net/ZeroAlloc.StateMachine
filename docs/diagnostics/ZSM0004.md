# ZSM0004 — Concurrent Mode Not Supported on Structs

**Severity:** Error

`Concurrent = true` was specified on a `partial struct`. Concurrent mode uses `Interlocked.CompareExchange` which requires a stable memory address (`ref long`). A struct does not guarantee this — it can be copied by value, invalidating the `ref`. The generator refuses to emit concurrent code for a struct.

## Example

```csharp
// ZSM0004: LightSwitch is a struct, Concurrent = true is not allowed
[StateMachine(InitialState = nameof(State.Off), Concurrent = true)]
[Transition<State, Trigger>(From = State.Off, On = Trigger.Flip, To = State.On)]
[Transition<State, Trigger>(From = State.On,  On = Trigger.Flip, To = State.Off)]
public partial struct LightSwitch { }
```

## How to fix

**Option 1 — change to a class:**

```csharp
public partial class LightSwitch { }  // class + Concurrent = true — supported
```

**Option 2 — remove concurrent mode:**

```csharp
[StateMachine(InitialState = nameof(State.Off), Concurrent = false)]
public partial struct LightSwitch { }  // struct without concurrent — supported
```

Use the struct option when you do not need thread safety and want to eliminate the instance heap allocation entirely. Use the class option when you need concurrent callers.
