# ZSM0003 — Single-Use Trigger

**Severity:** Warning

A trigger value appears in exactly one transition while other triggers are used in multiple transitions. This is a common source of typos — the same trigger accidentally declared under two different names.

## Example

```csharp
// ZSM0003: R.Back appears in only one transition
[StateMachine(InitialState = nameof(S.A))]
[Transition<S, R>(From = S.A, On = R.Go,   To = S.B)]
[Transition<S, R>(From = S.B, On = R.Go,   To = S.C)]
[Transition<S, R>(From = S.C, On = R.Back, To = S.A)]
public partial class MyMachine { }
```

## How to fix

Verify `R.Back` is not a typo for `R.Go`. If the single-use trigger is intentional, suppress the warning:

```csharp
#pragma warning disable ZSM0003
[Transition<S, R>(From = S.C, On = R.Back, To = S.A)]
#pragma warning restore ZSM0003
```
